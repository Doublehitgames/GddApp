# dataSchema

Define **pares clave-valor** con un tipo (int, float, seconds, percent, boolean, string) vinculados a la página. Es el "addon comodín" — cuando ninguno de los addons especializados cubre lo que necesitas, este lo hace.

## Por qué importa

El game design tiene muchas situaciones de "necesito anotar X aquí": rareza del arma, nivel requerido, nombre de animación, número mágico de balance. Sin un data schema, esos campos se convierten en strings sueltos dentro de un `richDoc` — sin validación, sin export estructurado. Aquí obtienes **tipo + min/max + referencias a otros addons** sin tener que crear un nuevo tipo de addon desde cero.

## Dónde suele vivir

- Páginas en blanco cuando quieres agregar campos tipados libres.
- Páginas tipadas cuando necesitas campos extra que el addon principal no cubre.

Singleton — 1 schema por página, con **N entradas**.

## Ejemplo

Schema con 3 campos: rareza (string), nivel requerido (int), fecha de descubrimiento (string):

```json
{
  "id": "schema-meta",
  "name": "Metadata",
  "entries": [
    {
      "id": "f1",
      "key": "rarity",
      "label": "Rareza",
      "valueType": "string",
      "value": "epic"
    },
    {
      "id": "f2",
      "key": "required_level",
      "label": "Nivel Requerido",
      "valueType": "int",
      "value": 25,
      "min": 1,
      "max": 100,
      "unit": "nv"
    },
    {
      "id": "f3",
      "key": "discovered_in",
      "label": "Descubierto en",
      "valueType": "string",
      "value": "Parche 1.4"
    }
  ]
}
```

## Referencias computadas

Los campos `economyLinkRef`/`economyLinkField` y `productionRef`/`productionField` permiten **valores derivados**: el panel muestra el valor actual del otro addon en lugar de que lo escribas manualmente. Útil para reflejar el "precio de compra" de un economyLink en el schema sin duplicar el número.

Cuando el addon fuente cambia, el valor lo sigue — si el economyLink cambia de 100 a 120, el schema lo refleja automáticamente.

<Callout type="tip" title="Cuándo NO usar esto">
  Si lo que necesitas ya tiene un addon dedicado (atributo, costo, modificador), usa ese
  addon específico en lugar de modelarlo con data schema. El data schema es un comodín para
  conceptos que no encajan en ninguno de los otros 15 addons.
</Callout>
