# attributeProfile

Asigna **valores específicos** de los atributos definidos en [`attributeDefinitions`](/docs/referencia/addons/attribute-definitions) a un personaje, clase o ítem.

## Por qué importa

Las definiciones dicen *"existen HP, ATK y DEF"*. Los perfiles dicen *"el Mago Oscuro tiene HP=80, ATK=15, DEF=3"*. Sin perfiles tendrías que copiar la estructura de estadísticas en cada página, y cualquier cambio en la definición (renombrar `atk` a `attack`) rompería todo.

## Dónde suele vivir

- Una página de tipo `👤 Personajes` (viene con profile + xpBalance + progressionTable).
- Páginas en blanco cuando necesitas estadísticas independientes (ej.: un NPC genérico, una base de enemigo).

Singleton — 1 perfil por página. Para múltiples personajes, crea múltiples páginas.

## Ejemplo

Perfil para un personaje:

```json
{
  "id": "profile-mage",
  "name": "Mago Oscuro",
  "definitionsRef": "section-attrs-base",
  "values": [
    { "id": "v1", "attributeKey": "hp",  "value": 80 },
    { "id": "v2", "attributeKey": "atk", "value": 15 },
    { "id": "v3", "attributeKey": "def", "value": 3 }
  ]
}
```

El `definitionsRef` apunta a la sección que contiene `attributeDefinitions`. El panel solo te permite elegir claves que existan allí — no puedes agregar accidentalmente un valor para un atributo inexistente.

## Validación de referencias cruzadas

Si renombras o eliminas una clave en `attributeDefinitions`, cualquier perfil que usaba esa clave:

- permanece visible en el panel con una advertencia de "clave huérfana"
- debe corregirse manualmente (elige una nueva clave o elimina la entrada)
- **no** rompe el renderizado — el addon degrada correctamente

## Wizard

El tipo de página `👤 Personajes` crea automáticamente:

1. Este `attributeProfile` (vacío, con `definitionsRef` ya completado)
2. Un `xpBalance` (curva de XP)
3. Una `progressionTable` (valores por nivel)
4. Un `attributeModifiers` (efectos específicos del personaje)

Todo vinculado al mismo `attributeDefinitions` que elegiste en el diálogo del wizard.

<Callout type="tip" title="Valores por defecto automáticos">
  Cuando creas el perfil mediante el wizard, ya viene con **todos los
  atributos de la definición** completados con su `defaultValue`.
  Solo necesitas editar lo que difiere del valor por defecto.
</Callout>
