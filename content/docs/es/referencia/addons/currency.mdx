# currency

Define una **moneda** en tu juego. Cada moneda tiene su propia página — no agrupas "oro + gemas + tickets" en el mismo addon. Por diseño, una moneda vive en una página.

## Por qué importa

Las monedas son referenciadas por muchos otros addons (precios de ítems, costos de habilidades, tasas de cambio, recompensas de recetas). Sin un addon dedicado, cada referencia sería un string suelto como `"oro"` — y cuando renombras la moneda a "monedas" todo el juego se rompe. Defínela aquí una sola vez y las referencias validadas hacen el resto.

## Dónde suele vivir

- Una página de tipo `🪙 Economía`. Wizard para Oro, Gemas, Tickets, etc.
- Páginas en blanco cuando quieres agrupar monedas atípicas.

Singleton — 1 moneda por página. Para múltiples monedas, crea múltiples páginas (una por moneda).

## Ejemplo

```json
{
  "id": "currency-gold",
  "name": "Oro",
  "code": "GOLD",
  "displayName": "Oro",
  "kind": "soft",
  "decimals": 0,
  "notes": "Moneda principal. Se gana en misiones, drops de enemigos, vendido en tienda."
}
```

## Quién apunta a esta moneda

- **`economyLink`** — precios de compra/venta en ítems.
- **`skills.SkillCost`** — cuando el tipo de costo de habilidad es `currency`.
- **`currencyExchange.CurrencyExchangeEntry`** — `fromCurrencyRef`/`toCurrencyRef` apuntan a monedas.
- **`craftTable.unlock.currency`** — desbloquear recetas/habilidades mediante un costo de moneda.

## Wizard

El tipo de página `🪙 Economía` crea este addon vacío con un relleno amigable:

- `code: "COINS"` (renombra según necesites)
- `displayName: "Coins"`
- `kind: "soft"`
- `decimals: 0`

Solo edítalo para que refleje la moneda real de tu juego.

<Callout type="tip" title="Convención de nombres">
  Usa <code>code</code> en UPPER_SNAKE para que sea un identificador estable en
  el JSON. Recomiendo prefijar las monedas premium (<code>GEM_PREMIUM</code>) y
  las de eventos (<code>EVENT_HALLOWEEN_TICKET</code>) — facilita filtrarlas en el motor.
</Callout>
