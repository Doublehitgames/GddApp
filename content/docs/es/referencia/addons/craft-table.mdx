# craftTable

Modela una **estación de producción** (forja, banco de trabajo, laboratorio de alquimia) que agrupa **múltiples recetas existentes**, cada una con su propia condición de desbloqueo (nivel, costo de moneda, ítem requerido).

## Por qué importa

En juegos con crafting maduro (Minecraft, Terraria, Stardew), las recetas no flotan sueltas — pertenecen a estaciones. Una "Forja" puede hacer espadas, hachas y armaduras. Un "Banco de trabajo" solo puede procesar madera. Sin este addon tendrías que modelar esa relación a mano (una página de Forja con una lista plana de nombres de recetas), perdiendo las referencias validadas.

## Dónde suele vivir

- Una página de tipo `🛠️ Estación de Producción`. El wizard pregunta qué recetas asociar.

Singleton — 1 craftTable por página.

  { name: "id", type: "string", required: "Sí", description: "ID interno." },

### `CraftTableUnlock`

También se reutiliza en `skills.unlock`. Tres bloques opcionales que se combinan (todos deben cumplirse cuando están habilitados):

```ts
{
  level?: { enabled: bool, xpAddonRef?: sectionId, level?: number },
  currency?: { enabled: bool, currencyAddonRef?: sectionId, amount?: number },
  item?: { enabled: bool, itemRef?: sectionId, quantity?: number },
}
```

- **level**: el jugador debe estar en el nivel X de una curva de XP específica.
- **currency**: pagar X de una moneda para desbloquear permanentemente.
- **item**: debe tener X unidades de un ítem (puede consumirse o solo requerirse).

## Ejemplo

Una forja con 3 recetas — una desbloqueada desde el inicio, una por nivel, una por compra:

```json
{
  "id": "ct-forge",
  "name": "Forja del Herrero",
  "entries": [
    {
      "id": "e1",
      "productionRef": "section-recipe-iron-sword",
      "category": "Armas",
      "order": 1
    },
    {
      "id": "e2",
      "productionRef": "section-recipe-steel-sword",
      "category": "Armas",
      "order": 2,
      "unlock": {
        "level": {
          "enabled": true,
          "xpAddonRef": "section-xp-blacksmith",
          "level": 5
        }
      }
    },
    {
      "id": "e3",
      "productionRef": "section-recipe-mythril-sword",
      "category": "Armas",
      "order": 3,
      "hidden": true,
      "unlock": {
        "currency": {
          "enabled": true,
          "currencyAddonRef": "section-gem",
          "amount": 100
        },
        "item": {
          "enabled": true,
          "itemRef": "section-rare-recipe-paper",
          "quantity": 1
        }
      }
    }
  ]
}
```

La Espada de Mithril requiere 100 Gemas + 1 papel raro. Oculta por defecto.

## Wizard

El tipo de página `🛠️ Estación de Producción` abre un diálogo preguntando qué recetas (páginas de tipo `📜 Receta`) asociar con la estación. Puedes:

- **Seleccionar existentes** del proyecto (selección múltiple).
- **Omitir** y agregarlas manualmente después desde el panel.

Cada receta seleccionada se convierte en una nueva entrada con un `order` secuencial. Ajustas las condiciones de desbloqueo y la categoría después.

## Quién apunta a craftTable

Generalmente **nadie** — es el consumidor final. Las páginas de recetas son las hojas; CraftTable las agrega; el motor lee el JSON.

Puedes tener múltiples páginas de Estación de Producción apuntando a las mismas recetas (ej.: una receta de "Pan" agregada tanto en el Banco de Trabajo del Hogar como en el Horno del Panadero). No hay ninguna regla que lo impida.
