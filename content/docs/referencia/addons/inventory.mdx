# inventory

Define **como um item se comporta no inventário** do jogador: peso, pilha máxima, durabilidade, categoria, vínculo (none/on-pickup/on-equip), volume, e visibilidade no shop.

## Por que importa

Quase todo motor de RPG/RTS/survival precisa dessas mesmas informações por item — e quase todos as implementam à mão, item por item. Aqui você define uma vez via UI, exporta pro JSON, e o motor lê. Bonus: campos como `categoryLibraryRef` permitem reaproveitar uma taxonomia central de categorias (Field Library) em vez de digitar "Espada" 50 vezes.

## Onde costuma viver

- Página de tipo `🎒 Itens` ou `⚔️ Itens com efeito`. Default.
- Páginas em branco quando você quer modelar containers exóticos (carteira, sacola de fadas).

Singleton — 1 inventory por página de item.

## Exemplo

Espada de Ferro — equipável, vinculável ao pegar, não-consumível:

```json
{
  "id": "inv-iron-sword",
  "name": "Inventário",
  "weight": 5.0,
  "stackable": false,
  "maxStack": 1,
  "inventoryCategory": "Espada",
  "slotSize": 1,
  "hasDurabilityConfig": true,
  "durability": 100,
  "maxDurability": 100,
  "bindType": "onEquip",
  "showInShop": true,
  "consumable": false,
  "discardable": true
}
```

Poção de Vida — empilhável, sem durabilidade, consumível:

```json
{
  "id": "inv-hp-potion",
  "name": "Inventário",
  "weight": 0.2,
  "stackable": true,
  "maxStack": 99,
  "inventoryCategory": "Consumível",
  "slotSize": 1,
  "durability": 0,
  "bindType": "none",
  "showInShop": true,
  "consumable": true,
  "discardable": true
}
```

## categoryLibraryRef vs string

Use `inventoryCategory` (string solta) pra protótipos. Quando o jogo amadurecer e você tiver 30+ itens, migre pra `categoryLibraryRef` apontando pra uma página de [`fieldLibrary`](/docs/referencia/addons/field-library) — assim renomear "Espada" → "Lâmina" se reflete em todos os itens automaticamente.

## Wizard

Não tem wizard próprio. Seedado por:

- `🎒 Itens` (junto com economyLink)
- `⚔️ Itens com efeito` (junto com economyLink + attributeModifiers)

Os defaults do wizard são razoáveis pra "item genérico" — você ajusta os booleans (consumable, stackable, etc.) e tá pronto.

## Quem aponta pra inventory

- **`production.ingredients[].itemRef`** e **`production.outputs[].itemRef`** — receitas referenciam páginas de itens.
- **`craftTable.unlock.item.itemRef`** — receitas que precisam de item específico pra desbloquear.
- **`skills.unlock.item.itemRef`** — skills que precisam de item de catalisador.
- **`economyLink.producedItemRef`** — uma página produz outro item (galinha → ovo).
