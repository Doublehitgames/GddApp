# dataSchema

Define **pares chave-valor** com tipo (int, float, seconds, percent, boolean, string) que ficam vinculados à página. É o "addon coringa" — quando nenhum dos addons especializados cobre o que você precisa, este resolve.

## Por que importa

Game design tem muito caso "preciso anotar X aqui": rarity da arma, nível requerido, animação, número mágico de balanceamento. Sem o data schema, esses campos viram strings soltas no `richDoc`, sem validação, sem possibilidade de export estruturado. Aqui você ganha **tipo + min/max + ref a outros addons** sem precisar criar um novo tipo de addon.

## Onde costuma viver

- Páginas em branco quando você quer adicionar campos livres tipados.
- Páginas tipadas quando precisa de campos extras que o addon principal não cobre.

Singleton — 1 schema por página, com **N entries**.

## Exemplo

Schema com 3 campos: rarity (string), required level (int), discovery date (string):

```json
{
  "id": "schema-meta",
  "name": "Metadados",
  "entries": [
    {
      "id": "f1",
      "key": "rarity",
      "label": "Raridade",
      "valueType": "string",
      "value": "épico"
    },
    {
      "id": "f2",
      "key": "required_level",
      "label": "Nível requerido",
      "valueType": "int",
      "value": 25,
      "min": 1,
      "max": 100,
      "unit": "lv"
    },
    {
      "id": "f3",
      "key": "discovered_in",
      "label": "Descoberto em",
      "valueType": "string",
      "value": "Patch 1.4"
    }
  ]
}
```

## Refs computadas

Os campos `economyLinkRef`/`economyLinkField` e `productionRef`/`productionField` permitem **valores derivados**: o painel mostra o valor atual do outro addon em vez de você digitar. Útil pra refletir "preço de compra" da economyLink no schema sem duplicar o número.

Quando o addon-fonte muda, o valor "viaja" — se o economyLink mudar de 100 pra 120, o schema reflete automaticamente.

<Callout type="tip" title="Quando NÃO usar">
  Se o que você quer já tem addon dedicado (atributo, custo, modifier), use o addon
  específico em vez de modelar com data schema. O data schema é fallback pra
  conceitos que não cabem em nenhum dos 15 outros addons.
</Callout>
