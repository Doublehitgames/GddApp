# Keyboard Shortcuts

Shortcuts speed up your flow when you already know what you want to do. **Inside the app, press `?` on any screen** to open a floating version of this page — the modal reads the same data.

<Callout type="tip" title="Mac vs Windows/Linux">
  Where the table shows <strong>Ctrl</strong>, on macOS use <strong>⌘</strong> (Cmd) instead.
  The app detects your system automatically — the in-app modal already shows
  the right symbol.
</Callout>

## General

Work on any screen in the app.

| Shortcut | What it does |
|---|---|
| `Ctrl + K` | Open the **quick page search** — start typing any page name, it appears in seconds |
| `Ctrl + /` or `?` | Open this shortcut list in the floating modal |
| `Esc` | Close open dialogs, menus, modals, and popovers |

## Section page

Work while you are editing a page.

| Shortcut | What it does |
|---|---|
| `N` | Create a **new page** (quick modal — pick type + name + parent) |
| `Ctrl + I` | Add an **addon** to this page (opens the picker) |
| `Ctrl + D` | **Duplicate** the current page (creates a copy alongside it, with all addons) |
| `Ctrl + M` *(Win/Linux)* `⌘ + Shift + M` *(Mac)* | **Move** the section to another location in the tree |

<Callout type="warning" title="Ctrl+D inside the browser">
  In the browser, <code>Ctrl + D</code> normally means "bookmark page". The app
  captures this shortcut to duplicate — you'll see a confirmation dialog the
  first time, as per our policy of respecting native shortcuts.
  You can accept without worry.
</Callout>

## Flowchart (Mind Map)

Work inside the flowchart / mind map editor.

| Shortcut | What it does |
|---|---|
| `Ctrl + Z` | **Undo** last action |
| `Ctrl + Y` | **Redo** undone action |
| `Ctrl + C` | **Copy** selected blocks |
| `Ctrl + V` | **Paste** blocks from clipboard |
| `Ctrl + D` | **Duplicate** selected block |
| `Esc` | **Deselect** everything |

## App conventions

A few general rules that apply to all shortcuts:

- **Native browser shortcuts**: the app **respects** (or asks for confirmation before overriding) shortcuts like `Ctrl+F`, `Ctrl+S`, `Ctrl+P`. Check the project settings if you want to audit them.
- **Focus on an input**: most single-key shortcuts (like `N` or `?`) **do not fire** when you are typing in an input/textarea. Modifier shortcuts (`Ctrl+...`) do.
- **Modal open**: some shortcuts are suspended when a modal/dialog is on top — `Esc` closes it first.

## Suggest a shortcut

Missing one? Send your suggestion describing the scenario — which screen, which action, which key combination you'd like. New shortcuts go through a review to avoid conflicts with native browser shortcuts.
