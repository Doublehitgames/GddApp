# Addons

An **addon** is a semantic block inside a page: it has its own schema, editing panel, and contributes to the exported JSON. Most addons are singletons (1 per page). The exceptions are `richDoc` and `remote-config`.

Each addon below has its own reference page. The links take you there.

<Callout type="info" title="Singleton vs. multiple">
  Almost all addons allow only **1 instance per page**. This ensures that
  internal refs (Skills → Modifiers on the same page) always have a unique
  target. The exceptions are `richDoc` (you can break content into several
  blocks) and `remote-config` (a page can produce multiple JSON files).
</Callout>
